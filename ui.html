Skrew UI Guide

The client UI is an SFML desktop interface drawn from C++ code. The main game screen uses a fixed 1000 by 700 world view, so most positions are explicit coordinates inside Skrew/src/main.cpp.

How The UI Works

Menu and flow

AppState in main.cpp switches between the main menu, nickname entry, room select, waiting room, and active game table.

Game table

The playing screen draws three panels: opponents, table piles, and your hand. Opponent and self cards use the same CARD_SIZE, currently 96x132, so all players read at the same visual scale.

Interactions

Mouse hit boxes come from the same rectangle shapes used for drawing. Opponent card clicks use getOpponentCardBounds(); self card clicks use handBoxes.

Animations

Local swaps and remote player actions use ghost card rectangles. Movement is eased with easeInOut() so card motion starts and stops smoothly. Remote discards start from the deck pile when the draw/card-pile count changed, from the drawn slot when a pending drawn card was already visible, and from the opponent row for hand-origin actions.

Text safety

Labels and messages are fitted before drawing with setFittedText(), fitTextToWidth(), and fitTextToBox() to avoid overlapping names, hints, room rows, and status text.

Files Used

  • Skrew/src/main.cpp: Most UI layout, rendering, text fitting, gameplay click handling, and animations.
  • Skrew/include/MainMenu.h and Skrew/src/MainMenu.cpp: Main menu buttons and title screen.
  • Skrew/include/GameClient.h and Skrew/src/GameClient.cpp: Backend requests and game state refreshes used by the UI.
  • Skrew/include/TextureManager.h and Skrew/src/TextureManager.cpp: Card image loading from assets/cards.
  • Skrew/assets/cards/*.png: Card face and back images.
  • Skrew/assets/fonts/arial.ttf: Font used by all SFML text.
  • Skrew/config.json: Backend host, port, and window size.

How To Edit It

  • Change card size in CARD_SIZE near the top of main.cpp. Player cards, opponent cards, piles, hit boxes, and animations follow this value.
  • Change spacing with PLAYER_CARD_SPACING, OPPONENT_CARD_SPACING, and OPPONENT_ROW_SPACING.
  • Move the table piles by editing deckPos, discardPos, and drawnPos inside runApp().
  • Move your hand by editing the handBoxes loop in runApp().
  • Move opponent cards by editing getOpponentCardBounds().
  • Adjust colors with the constants near the top of main.cpp, such as TABLE_PANEL, TEXT_PRIMARY, and ACCENT_GOLD.
  • Adjust remote action feel with remoteAnimDuration and easeInOut().
  • Adjust remote action source detection near the polling block that compares previousState and refreshedState. Deck-origin discards depend on draw/card-pile count fields such as draw_pile_count, deck_count, or cards_pile_count.
  • Prevent text overlap by using setFittedText(), fitTextToWidth(), or fitTextToBox() before drawing text.
Keep the SFML view at 1000 by 700 unless you also update the hard-coded panel and card positions. The configured window size can change, but the UI coordinates are still drawn in that fixed view.
